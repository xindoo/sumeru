## 网文逻辑审查

### 触发关键词
帮我检查下小说有没有bug、看看时间线有没有矛盾、人物有没有OOC、找剧情前后冲突、梳理伏笔有没有回收、检查小说剧情合理性、看看有没有剧情漏洞、人物行为不符合性格、检查时间线对不对、找小说前后矛盾的地方、帮我梳理所有伏笔、小说剧情bug检查、逻辑漏洞排查、小说剧情审查

### 核心功能

#### 两阶段审查流程设计

为避免一次性读入过多文本导致上下文超长，同时减少次要信息干扰，本 skill 采用两阶段审查流程：

**第一阶段：章节概要生成（Agent Team 并行）**
- 使用多 Agent 并行处理，每个 Agent 负责一定数量的章节
- 为每章生成核心剧情概要（去掉细节，仅保留关键事件）
- 概要写入 `.sumeru/review/summaries/` 目录
- 支持断点续传，已生成概要的章节可跳过

**第二阶段：逻辑审查（基于概要）**
- 读取所有章节概要，进行剧情连贯性和时间线审查
- 基于概要和大纲进行 OOC、伏笔、逻辑漏洞检查
- 如需深入检查特定章节，再加载完整章节内容

#### 功能列表

1. **时间线校验**：梳理全故事时间线，检查时间前后矛盾
   - 绝对时间校验：年份、月份、日期、节日的前后一致性
   - 相对时间校验：事件间隔、时长描述、成长周期的合理性
   - 季节气候校验：季节更迭、天气变化与时间线的匹配
   - 年龄时间线：人物年龄随剧情推进的自然增长
   - 事件时序：关键事件的先后顺序与因果关系

2. **剧情连贯性校验**：排查情节前后冲突
   - 设定一致性：世界观、力量体系、规则设定的前后统一
   - 物品状态：重要物品的获得、使用、丢失、状态变化追踪
   - 信息传递：人物已知信息与未知信息的边界维护
   - 地点空间：地理设定、场景转换、空间距离的合理性

3. **逻辑漏洞检查**：多维度排查故事逻辑问题
   - 因果逻辑：事件起因-经过-结果的合理性
   - 动机逻辑：人物行为动机的充分性与一致性
   - 能力逻辑：能力强度、使用限制、成长曲线的合理性
   - 资源逻辑：财物、物资、人力等资源的获取与消耗平衡
   - 社会逻辑：社会结构、组织运行、人际关系的合理性
   - 常识逻辑：是否违背基本常识、物理规律、生物规律

4. **OOC检测**：检查人物行为逻辑一致性
   - 性格一致性：核心性格特质的稳定性（可成长但不突变）
   - 价值观一致性：道德观念、行事原则的连贯性
   - 能力边界：技能水平、智力水平、体力水平的一致表现
   - 语言风格：口头禅、说话方式、用词习惯的统一
   - 行为模式：决策方式、应对策略、习惯性动作的一致性
   - 成长曲线：人物转变的渐进性与合理性（需有足够铺垫）

5. **伏笔追踪**：系统化识别与追踪伏笔
   - 伏笔标记：记录所有伏笔的设置位置、内容、暗示方向
   - 伏笔分类：按类型分类（剧情伏笔、人物伏笔、设定伏笔、情感伏笔）
   - 回收状态：标记已回收、待回收、疑似废弃的伏笔
   - 回收质量：评估伏笔回收的自然度、满意度、惊喜度
   - 伏笔建议：为待回收伏笔提供合理的回收方向建议

6. **冲突点梳理**：系统整理故事中的冲突
   - 冲突列表：按时间顺序列出所有冲突事件
   - 冲突类型：分类（人与人、人与环境、人与自我、阵营冲突等）
   - 冲突强度：评估冲突的紧张程度与重要性
   - 冲突解决：记录冲突的解决方式、结果与遗留问题
   - 冲突脉络：梳理主线冲突与支线冲突的关联与演进

7. **主线支线梳理**：确保叙事清晰不混乱
   - 主线脉络：核心剧情线的发展轨迹
   - 支线脉络：各支线剧情的独立发展与主线关联
   - 线索交织：多线索并行的合理性与清晰度
   - 叙事节奏：情节密度、高潮分布、节奏把控评估

8. **细纲一致性校验**：对比实际章节与原始细纲的偏差
   - 核心事件是否按细纲执行
   - 人物出场是否符合细纲规划
   - 爽点/悬念是否按预期设置
   - 识别遗漏的伏笔或新增的非规划内容

### 核心参数
| 参数 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `--only-summary` | boolean | 仅执行第一阶段，生成章节概要 | `--only-summary` |
| `--skip-summary` | boolean | 跳过第一阶段，直接使用已有的概要进行审查 | `--skip-summary` |
| `--agent-count` | number | 并行生成概要的 Agent 数量（默认 3） | `--agent-count 5` |
| `--task-split` | string | 任务分配策略：round-robin/smart/chunk（默认 round-robin） | `--task-split chunk` |
| `--full-chapters` | string | 指定需要加载完整内容审查的章节（逗号分隔） | `--full-chapters 1,5,10` |
| `--only` | string | 仅检查指定问题类型：timeline,ooc,plot,foreshadow,common | `--only timeline,ooc` |
| `--all` | boolean | 审查全部章节 | `--all` |
| `--dir` | string | 指定章节文件目录 | `--dir ./my-novel/chapters` |

### 输出内容
- **完整故事时间线图谱**：按时间轴整理的关键事件序列
- **时间线校验报告**：时间矛盾点、时长问题、时序错误清单
- **逻辑漏洞清单**：按维度分类的逻辑问题列表（含严重程度分级）
- **OOC检测报告**：人物行为异常点、性格偏离分析
- **伏笔追踪表**：伏笔设置位置、内容、回收状态、建议回收方向
- **冲突点列表**：全故事冲突事件汇总表（含类型、强度、解决状态）
- **剧情连贯性评分**：多维度综合评分（时间线、逻辑性、人物一致性等）
- **优化建议**：剧情调整方案、伏笔回收建议、冲突优化建议

### 数据持久化
**用户可见输出（当前工作目录）**：
- `剧情审查报告.md`：完整审查结果报告，包含所有问题、严重程度、修复建议（直接可读）

**中间数据（仅系统内部使用，存于`.sumeru/review/`目录）**：
- `summaries/`：章节概要目录（第一阶段生成）
  - `chapter-1.json`、`chapter-2.json`... 每章的核心剧情概要
  - `summary-progress.json`：概要生成进度记录
- `timeline.json`：完整时间线图谱，按时间轴排列的关键事件序列
- `issues.json`：问题清单，按严重程度、类型分类的所有问题记录（结构化数据）
- `foreshadowing-tracking.json`：伏笔追踪表，包含所有伏笔的位置、内容、回收状态
- `coherence-score.json`：剧情连贯性评分明细

#### 与其他 Skill 配合
- **前置 Skill**：读取 `sumeru-outline` 的大纲数据和 `sumeru-write` 的章节数据
  - 使用 `.sumeru/outline/` 的世界观、人设、大纲作为基准
  - **使用 `.sumeru/outline/chapter-outlines.json` 作为预期剧情参考**
  - 使用 `chapters/` 目录下的章节内容进行审查
  - 将实际章节内容与细纲进行对比，识别剧情偏离
- **后续 Skill**：生成的 `issues.json` 可被 `sumeru-write` 用于返工修改

#### 数据复用
- 返工修改时直接读取问题清单定位需要调整的章节
- 支持增量审查，新增章节时基于已有审查结果只检测新增内容
- 修复完成后可再次调用自动验证问题是否解决
